# RuLocalization — документация (состояние на текущую версию)

Мод **Russian Localization / Русификация** (`RuLocalization`) для **Caves of Qud** подменяет отображаемый в игре английский текст на русский через словарь, кэш и опционально машинный перевод (MT). Часть контента задаётся **данными** (XML в папке мода), часть — **рантайм-патчами Harmony** к сборке игры `Assembly-CSharp`.

---

## 1. Технологии и зависимости

| Компонент | Назначение |
|-----------|------------|
| **C# / .NET Standard 2.0** | Целевая платформа модов CoQ (см. шаблон `Mods.csproj` в игре). |
| **HarmonyLib (0Harmony)** | Патчи методов без изменения `Assembly-CSharp.dll`. Игра сама вызывает `ApplyHarmonyPatches` для сборок модов. |
| **Newtonsoft.Json** | Загрузка `config.json`, глоссариев, кэша переводов. |
| **System.Net (HttpWebRequest)** | Запросы к провайдерам перевода (Google и др.). |
| **XRL API** | `ModManager`, `MetricsManager`, типы `XRL.Messages.MessageQueue`, `XRL.UI.Popup`, `XRL.World.GameObject`, `XRL.Language.Grammar` и т.д. |

Сборка мода должна ссылаться на `Assembly-CSharp`, `0Harmony` и зависимости, как в официальном шаблоне модов.

## 2. Расположение и структура файлов

Путь по умолчанию:

`%USERPROFILE%\AppData\LocalLow\Freehold Games\CavesOfQud\Mods\RuLocalization\`

```
RuLocalization/
├── manifest.json          # Метаданные мода для игры (ID, заголовок, версия)
├── config.json            # Настройки движка перевода (провайдер, флаги, задержки)
├── Bootstrap.cs           # Точка входа (скомпилировано в DLL)
├── Genotypes.xml          # Слияние с базой: генотипы, подписи, extrainfo
├── EmbarkModules.xml      # Слияние: экраны создания персонажа, режимы, заголовки окон
├── Cache/
│   └── translations.json  # Персистентный кэш MT (ключ → перевод)
├── Translations/
│   ├── ui_strings.json    # Точные совпадения UI-фраз
│   ├── names.json         # Морфология имён (падежи, род, мн. число)
│   ├── patterns.json      # Префиксные замены (длинные ключи важнее)
│   ├── messages.json      # Доп. префиксные замены для сообщений
│   └── descriptions.json  # Фразы описаний (как словарь точных фраз)
├── Engine/                # Движок, провайдеры, грамматика, разбор тегов
├── Patches/               # Harmony-патчи
├── OptionalContent/       # Подсказки по ручному XML
├── API_POLICY_NOTE.txt    # Краткое напоминание про политику API
└── DOCUMENTATION.md       # Этот файл
```

Игра может предупреждать о `config.json` как о «старом» способе конфигурации мода; для **RuLocalization** это **отдельный** файл настроек движка, не замена `manifest.json`.

## 3. Инициализация

1. Игра загружает сборку мода и вызывает помеченный атрибутом **`[ModSensitiveCacheInit]`** метод `Bootstrap.Initialize()` в классе с **`[HasModSensitiveStaticCache]`** (паттерн CoQ для модов с кэшем).

2. **`Bootstrap.Initialize()`**:
   - получает путь к моду через `ModManager.GetMod(typeof(Bootstrap).Assembly)`;
   - вызывает **`TranslationEngine.Instance.Initialize(mod.Path)`** (конфиг, глоссарии, кэш, провайдер, фоновый поток);
   - при наличии **`mod.Harmony`** регистрирует **ручные** патчи попапов: **`ManualPopupRegistration.Register`**;
   - ошибки ручных патчей логируются и **не** обрывают работу движка.

3. **Автоматические** патчи (классы с `[HarmonyPatch]` в той же сборке) подхватываются игрой через **`Harmony.PatchAll`** при стандартной загрузке мода (отдельно от `Bootstrap`).

## 4. Движок перевода (`TranslationEngine`)

### 4.1. Роли

- Хранить **кэш** переводов в памяти и в `Cache/translations.json`.
- Загружать **статические глоссарии** из `Translations/*.json`.
- Вызывать **провайдер MT** (`ITranslationProvider`) при необходимости.
- Ограничивать **глубину рекурсии** при переводе имён (`ThreadStatic` счётчик).
- Учитывать **тип текста** через перечисление **`TranslationKind`**: `Message`, `Popup`, `Description` — и флаги в конфиге (`translate_messages`, `translate_popups`, `translate_descriptions`).

### 4.2. Порядок обработки строки в `Translate(text, kind)`

1. Пустая/слишком короткая строка → без изменений.
2. Если для данного `kind` перевод отключён в конфиге → без изменений.
3. **`IsNonTranslatable`** — эвристика (например, уже кириллица, нет букв, слишком длинная строка).
4. **`TryUiOrExactPhrase`** — поиск в `ui_strings.json` и `descriptions.json` (с учётом снятия цветовых тегов для ключа).
5. **Кэш** по нормализованному ключу (после `ColorTagParser.StripTags`).
6. Если включён **немедленный MT** для данного вида (`popup_use_immediate_translate` / `message_use_immediate_translate`) и провайдер задан — синхронный вызов API, запись в кэш.
7. Иначе — **префиксные замены** из `patterns.json` / `messages.json`, постановка в **очередь** фонового потока для MT.

### 4.3. Имена объектов (`TranslateName`)

- Используется из патчей отображаемых имён.
- Сначала **`names.json`** (`NameEntry`) и перенос цветовых тегов.
- Иначе — кэш и MT по общим правилам.

### 4.4. Фоновый поток

- Обрабатывает очередь **`PendingTranslation`** пакетами (`batch_size`, пауза `batch_delay_ms`).
- Периодически сохраняет кэш на диск.

### 4.5. Разметка CoQ (`ColorTagParser`)

Строки могут содержать конструкции вроде `{{W|текст}}`, префиксы `&y`, `^r` и т.д. Перед поиском в словаре и перед MT выполняется **снятие тегов**; после перевода — попытка **вернуть** обёртки через `TransferTags`.

## 5. Конфигурация (`config.json`)

| Поле | Смысл |
|------|--------|
| `provider` | `google` (по умолчанию), `deepl`, `openai`, `none` / `disabled`. |
| `api_key` | Ключ для DeepL/OpenAI. |
| `openai_model` | Модель OpenAI. |
| `batch_size`, `batch_delay_ms` | Параметры фоновой очереди. |
| `translate_messages` | Перевод строк лога/сообщений. |
| `translate_popups` | Перевод попапов (через патчи `Popup`). |
| `translate_descriptions` | Учёт kind `Description` в движке. |
| `translate_names` | Перевод имён через патчи `GameObject` + `TranslateName`. |
| `popup_use_immediate_translate` | Синхронный MT для попапов при первом показе. |
| `message_use_immediate_translate` | Синхронный MT для сообщений при первом показе. |
| `api_min_delay_ms` | Пауза между вызовами API (и в воркере, и при немедленном переводе). |
| `log_untranslated` | Логировать образцы непереведённого текста (отладка). |

При отсутствии файла движок может создать шаблон конфига.

## 6. Провайдеры машинного перевода (`ITranslationProvider`)

Реализации в папке `Engine/`:

| Класс | Особенности |
|-------|-------------|
| **GoogleTranslateProvider** | Неофициальный endpoint `translate.googleapis.com`, `client=gtx`, без API-ключа. В статическом конструкторе: TLS 1.2, **глобальный** `ServerCertificateValidationCallback`, принимающий любой сертификат — это обход проблем доверия сертификатов в среде Unity/Mono; с точки зрения безопасности это ослабление проверки TLS. |
| **DeepLProvider** | Требует `api_key` в конфиге. |
| **OpenAIProvider** | Требует ключ и модель. |

Условия использования внешних API и лимиты — на стороне поставщика; см. **`API_POLICY_NOTE.txt`**.

## 7. Harmony-патчи

Имена параметров в префиксах/постфиксах должны совпадать с **реальными** именами в IL или использоваться **позиционные** имена Harmony (`__0`, `__1`, `__result`), иначе `PatchAll` падает и **все** авто-патчи мода могут не примениться.

### 7.1. Автоматически (`[HarmonyPatch]` + `PatchAll`)

| Файл | Цель | Поведение |
|------|------|-----------|
| **MessageQueuePatch.cs** | `MessageQueue.Add` | Префикс: первый строковый аргумент (`__0`) прогоняется через `Translate(..., Message)`. |
| **DisplayNamePatch.cs** | Геттеры `GameObject`: `DisplayNameOnly`, `ShortDisplayName`, `BaseDisplayName`, `DisplayNameOnlyStripped` | Постфикс: при включённом `translate_names` — `TranslateName` для `__result`. |
| **GrammarPatch.cs** | `Grammar.Pluralize`, `Grammar.Cardinal` (int/long), `Grammar.Ordinal` (int/long) | Постфикс: множественное число через словарь имён; числительные и порядковые — через `RussianGrammar`. |

### 7.2. Ручная регистрация (`ManualPopupRegistration`)

Класс **`XRL.UI.Popup`**: методы ищутся рефлексией по имени и **первому параметру `string`**, чтобы не упираться в **неоднозначные перегрузки** (типичная ошибка — `AccessTools.Method` без списка типов).

Патчится первый строковый аргумент, перевод с kind **`Popup`**, для методов вроде:

- `ShowFail`, `ShowBlockSpace`, `ShowBlockPrompt`, `ShowBlockWithCopy`, `ShowYesNo`, `ShowYesNoCancel`, `AskString`, `ShowConversation`;
- все перегрузки **`ShowAsync`**, где первый параметр — `string`.

Попапы, у которых текст передаётся **не** первым аргументом или идёт через другие API, этим слоем **не** охватываются.

## 8. Данные XML (слияние с базой игры)

Игра подмешивает XML из папки мода к данным из `StreamingAssets/Base`.

| Файл | Назначение |
|------|------------|
| **Genotypes.xml** | `DisplayName`, блоки `extrainfo`, при необходимости удаление старых узлов через `Load="Remove"` (если поддерживается версией игры для данного узла). |
| **EmbarkModules.xml** | Заголовки окон создания персонажа, названия режимов, описания, тексты стартовых локаций и т.д. |

Так покрывается контент, который **не** проходит через `MessageQueue`/`Popup` (например, часть UI chargen).

Подробнее про ручное копирование больших XML — **`OptionalContent/README.txt`**.

## 9. JSON-глоссарии

| Файл | Формат | Использование |
|------|--------|----------------|
| **ui_strings.json** | `"английский ключ": "русский текст"` | Точное совпадение (и вариант без тегов). |
| **names.json** | Объект: ключ — исходное имя, значение — `NameEntry` (поля `nom`, `gen`, …, `plural_nom`, …) | Морфология и замена в `Pluralize`/`TranslateName`. |
| **patterns.json**, **messages.json** | Словарь строк | Подстановки по вхождению подстроки; порядок — после сортировки по длине ключа (длинные раньше). |
| **descriptions.json** | Словарь | Как дополнительный слой точных фраз для описаний. |

Дополнительно в каталоге `Cache/` могут подключаться файлы вида `manual_*.json` для ручного пополнения кэша.

## 10. Вспомогательная логика

- **`RussianGrammar`** — кардинальные и порядковые числительные и эвристики множественного числа для русского; для имён из словаря используются формы из `NameEntry`.
- **`NameEntry`** — DTO для `names.json`.

## 11. Диагностика

- Лог игры: **`%USERPROFILE%\AppData\LocalLow\Freehold Games\CavesOfQud\Player.log`**.
- Искать строки `RuLocalization`, `MODERROR`, `Harmony`, сообщения провайдера `RuLocalization Google:`.
- Типичные проблемы:
  - ошибка патча из-за **имени параметра** — не применяется весь блок авто-патчей;
  - **неоднозначная** перегрузка — использовать явный поиск метода или позиционные параметры;
  - MT не вызывается — проверить `provider`, сеть, кэш, флаги `translate_*` и immediate-флаги;
  - часть UI по-прежнему на английском — строка задаётся в **другом** кодовом пути (нужен новый патч или XML).

## 12. Ограничения текущей версии

- **Полный** охват игры одним MT и несколькими патчами **недостижим**: CoQ смешивает консольный UI, Kobold/UI Toolkit, прямые строки в коде и XML.
- Неофициальный Google Translate может **менять доступность**, лимиты и формат ответа.
- Глобальный обход проверки TLS в `GoogleTranslateProvider` снижает защиту от подмены трафика — осознанный компромисс для работы в Mono.
- Машинный перевод даёт **ошибки смысла**, особенно в лоре, квестах и терминах.
- Патчи **привязаны к версии игры**: смена сигнатур в `Popup`, `MessageQueue`, `Grammar` потребует обновления мода.

## 13. Направления дальнейшей работы

1. **Покрытие UI** — по логам и скриншотам находить классы/методы, куда не доходит `Popup`/`MessageQueue`, и добавлять узкие Harmony-патчи или дополнять XML.
2. **Словари** — расширять `ui_strings.json`, `names.json`, тематические `patterns.json` для стабильных формулировок без MT.
3. **Статический контент** — по желанию копировать и переводить куски `StreamingAssets/Base` (см. `OptionalContent/README.txt`).
4. **Провайдеры** — перейти на официальный API перевода с ключом, если появится бюджет и требования по ToS.
5. **Качество** — пост-редактура кэша (`Cache/translations.json`) или выгрузка в CSV для правки вручную.
6. **Совместимость** — после обновления игры прогонять тест: загрузка мода, отсутствие `HarmonyException` в логе, смоук-тест chargen и боя.
7. **manifest / конфиг CoQ** — при желании вынести настройки в [официальную схему конфигурации мода](https://wiki.cavesofqud.com/wiki/Modding:Mod_Configuration), сохранив отдельный файл настроек движка под другим именем, чтобы не смешивать с `config.json` движка перевода.
